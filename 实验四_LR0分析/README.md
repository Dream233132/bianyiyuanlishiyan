# 实验四：LR(0)分析法

## 实验目的
实现LR(0)语法分析算法，自动生成LR(0)项目集族和分析表，并给出语句的分析过程。

## LR(0)分析原理

### 1. LR(0)项目
LR(0)项目是在产生式右部某个位置添加一个点的产生式。

例如，对于产生式 A -> XYZ，有4个LR(0)项目：
- A -> ·XYZ
- A -> X·YZ
- A -> XY·Z
- A -> XYZ·

### 2. 项目集闭包 CLOSURE
对于项目集 I，CLOSURE(I) 的计算：
1. I 中的所有项目都属于 CLOSURE(I)
2. 如果 A -> α·Bβ 在 CLOSURE(I) 中，且 B -> γ 是一个产生式，则将 B -> ·γ 加入 CLOSURE(I)
3. 重复步骤2，直到不再有新项目加入

### 3. 转换函数 GOTO
GOTO(I, X) 表示从项目集 I 经过符号 X 转换到的项目集：
1. 对于 I 中所有形如 A -> α·Xβ 的项目
2. 将 A -> αX·β 加入结果集
3. 对结果集求闭包

### 4. LR(0)项目集族构造
1. 初始项目集 I0 = CLOSURE({S' -> ·S})
2. 对每个项目集 I 和每个文法符号 X：
   - 如果 GOTO(I, X) 非空且不在已有项目集中，则加入
3. 重复步骤2，直到不再产生新项目集

### 5. LR(0)分析表
**ACTION表：**
- 如果 [A -> α·aβ] 在 Ii 中，且 GOTO(Ii, a) = Ij，则 ACTION[i, a] = sj（移进）
- 如果 [A -> α·] 在 Ii 中，则对所有终结符 a，ACTION[i, a] = rj（规约，j是产生式编号）
- 如果 [S' -> S·] 在 Ii 中，则 ACTION[i, $] = acc（接受）

**GOTO表：**
- 如果 GOTO(Ii, A) = Ij，则 GOTO[i, A] = j

### 6. LR(0)分析过程
使用状态栈和符号栈进行自底向上的语法分析：
1. 初始化：状态栈压入0
2. 重复以下步骤：
   - 查ACTION表，根据栈顶状态和当前输入符号决定动作
   - 移进：压入新状态和符号
   - 规约：弹出产生式右部长度的状态和符号，查GOTO表压入新状态
   - 接受：分析成功
   - 错误：分析失败

## 实验要求的文法

```
E -> aA | bB
A -> cA | d
B -> cB | d
```

**增广文法：**
```
0: E' -> E
1: E -> aA
2: E -> bB
3: A -> cA
4: A -> d
5: B -> cB
6: B -> d
```

## 文法格式

```
# 注释行以#开头
非终结符->产生式右部
```

**规则：**
- 非终结符用大写字母表示
- 终结符用小写字母表示
- 不支持空格（会被自动去除）
- 第一个产生式的左部是开始符号

## 实现特点

### 1. 自动构造项目集族
- 使用闭包算法
- 使用GOTO函数
- 自动识别相同项目集

### 2. 自动生成分析表
- 根据项目集构造ACTION表
- 根据GOTO函数构造GOTO表
- 检测移进-规约冲突和规约-规约冲突

### 3. 语法分析
- 使用双栈（状态栈和符号栈）
- 详细输出每一步
- 错误检测和报告

### 4. 冲突检测
- 移进-规约冲突：同一状态对同一符号既可移进又可规约
- 规约-规约冲突：同一状态对同一符号有多个规约动作

## LR(0)文法条件

一个文法是LR(0)的，当且仅当：
- 分析表中没有冲突项
- 每个项目集中最多有一个规约项目
- 如果有规约项目，则没有移进项目

## LR(0) vs LL(1)

| 特性 | LR(0) | LL(1) |
|------|-------|-------|
| 分析方向 | 自底向上 | 自顶向下 |
| 分析能力 | 较强 | 较弱 |
| 文法限制 | 较少 | 较多（需消除左递归） |
| 实现复杂度 | 较高 | 较低 |
| 分析表大小 | 较大 | 较小 |

## 常见问题

### 1. 文法不是LR(0)
如果出现冲突，可以考虑：
- 使用SLR(1)分析（增加Follow集判断）
- 使用LR(1)或LALR(1)分析
- 改写文法

### 2. 输入格式
- 符号不需要空格分隔
- 非终结符用大写字母
- 终结符用小写字母

### 3. 项目集数量
- 项目集数量取决于文法复杂度
- 本实验的文法生成10个项目集
