# 实验二：手动词法分析器

## 实验目的
利用状态转换图手工构造C--语言的词法分析器，不使用Flex等自动工具。

## 实现原理

### 状态转换图法
词法分析器使用有限状态自动机（DFA）实现，通过状态转换识别不同类型的单词符号。

### 主要状态转换

#### 1. 标识符识别状态图
```
状态0 --[字母/下划线]--> 状态1 --[字母/数字/下划线]--> 状态1
                                    |
                                    +--> 接受状态（标识符或关键字）
```

#### 2. 数字识别状态图
```
状态0 --[数字]--> 状态1 --[数字]--> 状态1
                    |
                    +--[.]--> 状态2 --[数字]--> 状态3
                                                  |
                                                  +--[e/E]--> 状态4 --[+/-]--> 状态5 --[数字]--> 状态6
```

#### 3. 运算符识别状态图
```
状态0 --[<]--> 状态1 --[=]--> 接受状态（<=）
                |
                +--> 接受状态（<）

状态0 --[>]--> 状态1 --[=]--> 接受状态（>=）
                |
                +--> 接受状态（>）

状态0 --[=]--> 状态1 --[=]--> 接受状态（==）
                |
                +--> 接受状态（=）
```

## 核心类和方法

### Lexer类
- **advance()**: 读取下一个字符，更新行号
- **peek()**: 向前看一个字符（不移动位置）
- **skipWhitespace()**: 跳过空白字符
- **skipComment()**: 跳过单行和多行注释
- **recognizeIdentifier()**: 识别标识符和关键字
- **recognizeNumber()**: 识别整数、浮点数、科学计数法
- **getNextToken()**: 获取下一个Token

### Token结构
```cpp
struct Token {
    TokenType type;   // 种别码
    string value;     // 单词值
    int line;         // 行号
};
```

## 实现特点

### 1. 状态机设计
- 使用switch-case实现状态转换
- 每个状态对应一个处理逻辑
- 通过peek()实现向前看功能

### 2. 关键字识别
- 先按标识符规则识别
- 再查关键字表判断是否为关键字
- 使用C++ map存储关键字表

### 3. 数字识别
- 支持整数：123
- 支持浮点数：3.14159
- 支持科学计数法：2.71828e10, 1.5E-5

### 4. 运算符识别
- 双字符运算符优先匹配（<=, >=, ==）
- 使用peek()向前看一个字符
- 单字符运算符直接识别

### 5. 注释处理
- 单行注释：// ...
- 多行注释：/* ... */
- 注释内容不输出

### 6. 错误处理
- 识别非法字符
- 输出错误行号和字符
- 继续分析后续内容

| 代码量 | 少 | 多 |

## 输出格式

与实验一相同，输出二元组格式：`(种别码, 值)`

示例：
```
(2, if)
(18, ()
(6, a)
(16, GT)
(6, b)
(19, ))
```
