# 编译原理实验代码

## 项目概述

本项目包含编译原理课程的四个实验，涵盖词法分析和语法分析的主要技术。所有实验均使用C/C++实现，包含详细注释和说明文档。

## 实验列表

### 实验一：Flex词法分析器
- **目录**: `实验一_Flex词法分析器/`
- **技术**: 使用Flex工具自动生成词法分析器
- **功能**: 识别C--语言的所有单词符号，输出二元组形式
- **特色**: 
  - 支持整数、浮点数、科学计数法
  - 支持单行和多行注释
  - 完整的错误检测

### 实验二：手动词法分析器
- **目录**: `实验二_手动词法分析器/`
- **技术**: 使用状态转换图手工实现词法分析器
- **功能**: 与实验一相同，但完全手动实现
- **特色**:
  - 深入理解词法分析原理
  - 状态机设计
  - 向前看（peek）机制

### 实验三：LL(1)分析法
- **目录**: `实验三_LL1分析/`
- **技术**: 自顶向下的语法分析
- **功能**: 
  - 自动计算First集和Follow集
  - 自动生成LL(1)分析表
  - 输出详细的分析过程
- **特色**:
  - 支持任意LL(1)文法
  - 冲突检测
  - 完整的分析过程展示

### 实验四：LR(0)分析法
- **目录**: `实验四_LR0分析/`
- **技术**: 自底向上的语法分析
- **功能**:
  - 自动构造LR(0)项目集族
  - 自动生成LR(0)分析表
  - 输出详细的分析过程
- **特色**:
  - 实现实验要求的文法
  - 双栈分析过程
  - 冲突检测

## 环境要求

## 快速开始

### 实验一和实验二（词法分析）

```bash
# 进入实验目录
cd 实验一_Flex词法分析器

# Windows环境
flex lexer.l
gcc lex.yy.c -o lexer.exe
lexer.exe test_correct.c

# Linux/Mac环境
make
make test_correct
```

### 实验三（LL(1)分析）

```bash
cd 实验三_LL1分析

# Windows环境
g++ -std=c++11 ll1_parser.cpp -o ll1_parser.exe
ll1_parser.exe grammar.txt "id + id * id"

# Linux/Mac环境
make
make test1
```

### 实验四（LR(0)分析）

```bash
cd 实验四_LR0分析

# Windows环境
g++ -std=c++11 lr0_parser.cpp -o lr0_parser.exe
lr0_parser.exe grammar.txt acccd

make test
```

## 测试说明

### 词法分析测试
- **test_correct.c**: 包含所有正确的C--语言结构
  - 测试所有关键字
  - 测试所有运算符（包括新增的减法、除法、大于、大于等于）
  - 测试整数、浮点数、科学计数法
  
- **test_error.c**: 包含多种词法错误
  - 非法字符：@, #, $, &, %, ^, ~, `, |
  - 用于测试错误检测能力

### 语法分析测试
- **LL(1)**: 提供算术表达式和if-else语句两种文法
- **LR(0)**: 使用实验要求的文法 E->aA|bB, A->cA|d, B->cB|d

## 输出格式

### 词法分析输出
```
(种别码, 值)
例如：
(2, if)
(18, ()
(6, x)
(16, GT)
(7, 10)
```

### 语法分析输出
包含以下部分：
1. 文法信息（产生式列表）
2. First集和Follow集（LL(1)）或项目集族（LR(0)）
3. 分析表
4. 详细的分析过程（栈、输入、动作）

## 实现要点

### 词法分析
1. **正则表达式**: 定义各类单词符号的模式
2. **优先级**: 双字符运算符优先于单字符运算符
3. **状态转换**: 手动实现使用状态机
4. **错误处理**: 识别并报告非法字符

### LL(1)分析
1. **First集计算**: 迭代算法，处理ε产生式
2. **Follow集计算**: 基于First集，迭代直到不变
3. **分析表构造**: 根据First和Follow集填表
4. **冲突检测**: 检查是否为LL(1)文法

### LR(0)分析
1. **闭包计算**: 添加点后非终结符的所有产生式
2. **GOTO函数**: 计算状态转换
3. **项目集族**: 使用队列构造所有项目集
4. **分析表**: 根据项目集构造ACTION和GOTO表
