# 实验三：LL(1)分析法

## 实验目的
实现LL(1)语法分析算法，自动生成First集、Follow集和LL(1)分析表，并给出语句的分析过程。

## LL(1)分析原理

### 1. First集
First(α) 是可以从符号串α推导出的所有串的首终结符集合。

**计算规则：**
- 如果 X 是终结符，First(X) = {X}
- 如果 X -> ε，则 ε ∈ First(X)
- 如果 X -> Y1Y2...Yk，则：
  - First(Y1) - {ε} ⊆ First(X)
  - 如果所有Yi都能推导出ε，则 ε ∈ First(X)

### 2. Follow集
Follow(A) 是可能在某个句型中紧跟在A后面的终结符集合。

**计算规则：**
- $ ∈ Follow(S)，S是开始符号
- 如果 A -> αBβ，则 First(β) - {ε} ⊆ Follow(B)
- 如果 A -> αB 或 A -> αBβ 且 ε ∈ First(β)，则 Follow(A) ⊆ Follow(B)

### 3. LL(1)分析表
对于产生式 A -> α：
- 对于 First(α) 中的每个终结符 a，将 A -> α 加入 M[A, a]
- 如果 ε ∈ First(α)，对于 Follow(A) 中的每个符号 b，将 A -> α 加入 M[A, b]

### 4. LL(1)分析过程
使用栈和分析表进行自顶向下的语法分析：
1. 初始化：栈中压入$和开始符号S
2. 重复以下步骤：
   - 如果栈顶是终结符，与输入匹配
   - 如果栈顶是非终结符，查分析表，将产生式右部逆序压栈
   - 直到栈空且输入结束

## 文法格式

```
# 注释行以#开头
非终结符 -> 产生式右部 | 候选式
```

**规则：**
- 非终结符用大写字母表示
- 终结符用小写字母或符号表示
- ε 表示空串
- 用 | 分隔多个候选式
- 第一个产生式的左部是开始符号

## 示例文法

### 文法1：算术表达式（消除左递归后）
```
E -> T E'
E' -> + T E' | ε
T -> F T'
T' -> * F T' | ε
F -> ( E ) | id
```

**原文法（左递归）：**
```
E -> E + T | T
T -> T * F | F
F -> ( E ) | id
```

### 文法2：简单语句文法
```
S -> a | ^ | ( T )
T -> S T'
T' -> , S T' | ε
```

其中：
- a = 原子符号
- ^ = 空符号
- ( T ) = 括号包围的列表
- , = 列表分隔符

## 实现特点

### 1. 自动计算First集
- 使用迭代算法
- 处理ε产生式
- 处理符号串的First集

### 2. 自动计算Follow集
- 基于First集计算
- 处理产生式右部的所有情况
- 迭代直到不再变化

### 3. 自动生成分析表
- 根据First集和Follow集构造
- 检测冲突（非LL(1)文法）
- 使用map存储稀疏表

### 4. 语法分析
- 使用栈模拟分析过程
- 详细输出每一步
- 错误检测和报告

## LL(1)文法条件

一个文法是LL(1)的，当且仅当：
1. 不含左递归
2. 对于每个非终结符A的任意两个不同产生式 A -> α | β：
   - First(α) ∩ First(β) = ∅
   - 如果 ε ∈ First(β)，则 First(α) ∩ Follow(A) = ∅

## 常见问题

### 1. 文法不是LL(1)
如果出现冲突，需要：
- 消除左递归
- 提取左公因子
- 改写文法

### 2. 输入格式
- 符号之间用空格分隔
- 非终结符用大写字母
- 终结符用小写字母或符号

### 3. ε的表示
在文法文件中使用 `ε` 表示空串
